# 02 - Workflow and Documentation

## Context management workflow
For each task:
1. Create a dedicated context file in `docs/`.
2. Track decisions, patterns, anti-patterns, and assumptions.
3. Update context incrementally as work evolves.
4. Cross-reference related context files.

## Documentation quality rules
- Justify architectural decisions.
- Record alternatives and why they were rejected.
- Capture reasoning evolution, not only final conclusions.
- Keep documentation useful for onboarding/debugging.

## Intent
Reduce knowledge loss and make future modifications consistent.
