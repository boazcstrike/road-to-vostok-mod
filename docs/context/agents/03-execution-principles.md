# 03 - Execution Principles

## 1) Think before coding
- State assumptions.
- Surface ambiguity and tradeoffs.
- Ask when unclear.

## 2) Simplicity first
- Build only what is requested.
- Avoid speculative abstractions/configurability.
- Prefer minimal, readable implementations.

## 3) Surgical changes
- Touch only necessary lines.
- Avoid unrelated refactors/formatting.
- Remove only unused code created by your own change.

## 4) Goal-driven execution
- Convert vague asks into verifiable outcomes.
- For multi-step work, define steps and checks.
- Verify behavior with tests/checks before claiming done.

## Success signal
Cleaner diffs, fewer rewrites, and early clarifying questions.
